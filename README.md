# ДЗ-3 Косарева Софья ББИ2505

Этот репозиторий содержит автоматизированную систему для генерации данных о жилье и их последующего анализа

* `generate.py`: создает `data.csv` с параметрами жилья (район, площадь, стоимость)
* `report.js`: считывает `data.csv` и формирует `report.html` с визуализацией

## Инструкция по запуску

Данный проект предназначен для запуска в среде GitHub Codespaces

1. **Запуск среды**: откройте проект в GitHub Codespaces (зеленая кнопка `Code` - `Codespaces` - `Create codespace on main`).
2. **Подготовка**: в терминале Codespaces сделайте скрипт исполняемым:
   ```bash
   chmod +x run.sh
3. **Генерация данных (создаст файл data/data.csv):**
    ```bash
    ./run.sh build_generator
    ./run.sh run_generator
4. **Анализ данных:**
    ```bash
    ./run.sh build_reporter
    ./run.sh run_reporter
5. **Просмотр отчёта:**
Запустите встроенный веб-сервер:
    ```bash
    ./run.sh report_server

После запуска Codespaces предложит открыть порт 8080 - нажмите "Open in Browser".
В открывшейся вкладке браузера кликните на файл report.html

### Дополнительный функционал
Выводит структуру всех файлов и директорий проекта начиная с текущей папки

```bash
./run.sh structure
```
Удаляет все сгенерированные данные — файлы `.csv` и `.html` из папки `data/`. После выполнения этой команды папка должна остаться пустой, а повторный запуск `run_generator` и `run_reporter` должен всё пересоздать.

```bash
./run.sh clear_data
```

Запускает контейнер генератора и выводит её содержимое data изнутри контейнера. Так можно убедиться, что контейнер видит данные на хосте.
```bash
./run.sh inside_generator
```

Аналогично предыдущей команде, но для контейнера аналитика. Проверяет, что контейнер reporter видит файлы data с хоста
```bash
./run.sh inside_reporter
```
